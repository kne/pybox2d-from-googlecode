Class Hierarchy

  • object: The most base type
    • Box2D.Box2D.b2AABB: An axis aligned bounding box.
    • Box2D.Box2D.b2AssertException: Proxy of C++ b2AssertException class
    • Box2D.Box2D.b2Body: A rigid body.
    • Box2D.Box2D.b2BodyDef: A body definition holds all the data needed to construct a rigid body.
    • Box2D.Box2D.b2BroadPhase: The broad-phase is used for computing pairs and performing volume queries and ray casts.
    • Box2D.Box2D.b2ClipVertex: Used for computing contact manifolds.
    • Box2D.Box2D.b2Color: Color for debug drawing.
    • Box2D.Box2D.b2Contact: The class manages contact between two shapes.
    • Box2D.Box2D.b2ContactEdge: A contact edge is used to connect bodies and contacts together in a contact graph where each body is a node and each contact is an edge.
    • Box2D.Box2D.b2ContactFeature: The features that intersect to form the contact point This must be 4 bytes or less.
    • Box2D.Box2D.b2ContactFilter: Implement this class to provide collision filtering.
    • Box2D.Box2D.b2ContactID: Proxy of C++ b2ContactID class
    • Box2D.Box2D.b2ContactImpulse: Contact impulses for reporting.
    • Box2D.Box2D.b2ContactListener: Implement this class to get contact information.
    • Box2D.Box2D.b2ContactManager: Proxy of C++ b2ContactManager class
    • Box2D.Box2D.b2ContactPoint: Proxy of C++ b2ContactPoint class
    • Box2D.Box2D.b2DebugDraw: Implement and register this class with a b2Worldto provide debug drawing of physics entities in your game.
      • Box2D.Box2D.b2DebugDrawExtended: Proxy of C++ b2DebugDrawExtended class
    • Box2D.Box2D.b2DestructionListener: Joints and fixtures are destroyed when their associated body is destroyed.
    • Box2D.Box2D.b2DistanceInput: Input for b2Distance.
    • Box2D.Box2D.b2DistanceOutput: Output for b2Distance.
    • Box2D.Box2D.b2DistanceProxy: A distance proxy is used by the GJK algorithm.
    • Box2D.Box2D.b2Filter: This holds contact filtering data.
    • Box2D.Box2D.b2Fixture: A fixture is used to attach a shape to a body for collision detection.
    • Box2D.Box2D.b2FixtureDef: A fixture definition is used to create a fixture.
    • Box2D.Box2D.b2FixtureProxy: This proxy is used internally to connect fixtures to the broad-phase.
    • Box2D.Box2D.b2Jacobian: Proxy of C++ b2Jacobian class
    • Box2D.Box2D.b2Joint: The base joint class.
      • Box2D.Box2D.b2DistanceJoint: A distance joint constrains two points on two bodies to remain at a fixed distance from each other.
      • Box2D.Box2D.b2FrictionJoint: Friction joint.
      • Box2D.Box2D.b2GearJoint: A gear joint is used to connect two joints together.
      • Box2D.Box2D.b2LineJoint: A line joint.
      • Box2D.Box2D.b2MouseJoint: A mouse joint is used to make a point on a body track a specified world point.
      • Box2D.Box2D.b2PrismaticJoint: A prismatic joint.
      • Box2D.Box2D.b2PulleyJoint: The pulley joint is connected to two bodies and two fixed ground points.
      • Box2D.Box2D.b2RevoluteJoint: A revolute joint constrains two bodies to share a common point while they are free to rotate about the point.
      • Box2D.Box2D.b2RopeJoint: A rope joint enforces a maximum distance between two points on two bodies.
      • Box2D.Box2D.b2WeldJoint: A weld joint essentially glues two bodies together.
    • Box2D.Box2D.b2JointDef: Joint definitions are used to construct joints.
      • Box2D.Box2D.b2DistanceJointDef: Distance joint definition.
      • Box2D.Box2D.b2FrictionJointDef: Friction joint definition.
      • Box2D.Box2D.b2GearJointDef: Gear joint definition.
      • Box2D.Box2D.b2LineJointDef: Line joint definition.
      • Box2D.Box2D.b2MouseJointDef: Mouse joint definition.
      • Box2D.Box2D.b2PrismaticJointDef: Prismatic joint definition.
      • Box2D.Box2D.b2PulleyJointDef: Pulley joint definition.
      • Box2D.Box2D.b2RevoluteJointDef: Revolute joint definition.
      • Box2D.Box2D.b2RopeJointDef: Rope joint definition.
      • Box2D.Box2D.b2WeldJointDef: Weld joint definition.
    • Box2D.Box2D.b2JointEdge: A joint edge is used to connect bodies and joints together in a joint graph where each body is a node and each joint is an edge.
    • Box2D.Box2D.b2Manifold: A manifold for two touching convex shapes.
    • Box2D.Box2D.b2ManifoldPoint: A manifold point is a contact point belonging to a contact manifold.
    • Box2D.Box2D.b2MassData: This holds the mass data computed for a shape.
    • Box2D.Box2D.b2Mat22: A 2-by-2 matrix.
    • Box2D.Box2D.b2Mat33: A 3-by-3 matrix.
    • Box2D.Box2D.b2Pair: Proxy of C++ b2Pair class
    • Box2D.Box2D.b2QueryCallback: Callback class for AABB queries.
    • Box2D.Box2D.b2RayCastCallback: Callback class for ray casts.
    • Box2D.Box2D.b2RayCastInput: Ray-cast input data.
    • Box2D.Box2D.b2RayCastOutput: Ray-cast output data.
    • Box2D.Box2D.b2Shape: A shape is used for collision detection.
      • Box2D.Box2D.b2CircleShape: A circle shape.
      • Box2D.Box2D.b2EdgeShape: A line segment (edge) shape.
      • Box2D.Box2D.b2LoopShape: A loop shape is a free form sequence of line segments that form a circular list.
      • Box2D.Box2D.b2PolygonShape: A convex polygon.
    • Box2D.Box2D.b2Sweep: This describes the motion of a body/shape for TOI computation.
    • Box2D.Box2D.b2TOIInput: Input parameters for b2TimeOfImpact.
    • Box2D.Box2D.b2TOIOutput: Proxy of C++ b2TOIOutput class
    • Box2D.Box2D.b2Transform: A transform contains translation and rotation.
    • Box2D.Box2D.b2Vec2: A 2D column vector.
    • Box2D.Box2D.b2Vec3: A 2D column vector with 3 elements.
    • Box2D.Box2D.b2Version: Version numbering scheme.
    • Box2D.Box2D.b2World: The world class manages all physics entities, dynamic simulation, and asynchronous queries.
    • Box2D.Box2D.b2WorldManifold: This is used to compute the current state of a contact manifold.
